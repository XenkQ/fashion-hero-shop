# Website Inspection Checklist

## Screenshots

- Desktop viewport around 1440px wide.
- Tablet viewport around 768px wide.
- Mobile viewport around 390px wide.
- Open menus, modals, dropdowns, hover states, loading states, and empty states when present.

## Design Tokens

Capture actual computed values for:

- Colors: background, text, muted text, primary actions, borders, overlays, error and success states.
- Typography: font family, size, weight, line height, letter spacing, text transform.
- Spacing: recurring padding, margin, gap, max-width, section rhythm.
- Shape: border radius, border width, shadows.
- Motion: transitions, keyframes, duration, easing.

## Component Specs

Each spec should include:

- Source URL and viewport.
- Target implementation file.
- Screenshot path.
- DOM structure summary.
- Exact text content.
- Asset paths or asset source URLs.
- Default, hover, focus, active, open, closed, loading, and empty states where relevant.
- Responsive behavior at desktop, tablet, and mobile.
- Interaction model: static, click-driven, hover-driven, scroll-driven, time-driven, or mixed.

## Asset Notes

List images, videos, fonts, icons, and background images. Note layered assets separately when one visual section uses multiple overlaid files.
