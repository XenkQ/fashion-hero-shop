# GitHub Pages Deployment Implementation Summary

## Implemented

- Updated `next.config.ts` for static export:
  - `output: "export"`
  - `trailingSlash: true`
  - production-only `basePath: "/fashion-hero-shop"`
  - `images.unoptimized: true`
  - `NEXT_PUBLIC_BASE_PATH` exposed for client/server code
- Added `src/lib/site-paths.ts` with `withSiteBasePath(src)` for local public image assets.
- Updated all `next/image` local public asset usages to pass image `src` values through `withSiteBasePath(...)`.
- Kept `Link` routes unchanged so Next applies `basePath` automatically.
- Made dynamic routes explicit static exports:
  - added `dynamicParams = false` to `src/app/products/[slug]/page.tsx`
  - added `dynamicParams = false` to `src/app/collections/[slug]/page.tsx`
- Removed server-side `searchParams` usage from the collection route.
- Added `src/app/collections/[slug]/collection-client-view.tsx` to read the optional `seller` query with `useSearchParams()` behind `Suspense`.
- Added `.github/workflows/deploy-pages.yml`:
  - deploys on pushes to `main`
  - supports manual `workflow_dispatch`
  - runs `npm ci`, `npm run lint`, and `npm run build`
  - adds `out/.nojekyll`
  - uploads `out/` as a Pages artifact
  - deploys with `actions/deploy-pages`

## Behavior

- Production builds export the app into `out/` for GitHub Pages.
- Static assets and public images are emitted with `/fashion-hero-shop/...` URLs in production output.
- Local development remains unprefixed because `basePath` and `NEXT_PUBLIC_BASE_PATH` are only set for production builds.
- Collection seller filtering still works through `?seller=...`, but query reading happens in a client component so the route can remain statically exported.

## Notes

- `npm run lint` is blocked by PowerShell execution policy on this machine; use `npm.cmd run lint`.
- The first local production build failed because the sandbox could not fetch `next/font/google` from Google Fonts. Re-running `npm.cmd run build` with network permission passed.
- `out/` is ignored by `.gitignore`; it is generated by CI and should not be committed.
- After pushing to `main`, GitHub Pages must be configured to use GitHub Actions as the Pages source in repository settings.
